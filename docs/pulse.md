## 情绪残留系统 · 框架

### 一、触发

```
任何驱力在30分钟滑动窗口内达到阈值 N 次 → 产生残留

"达到阈值" =
  对话中: 驱力 >= CONV_THRESHOLDS.get(k, 0.65)
  静默中: 驱力 >= BG_THRESHOLDS_MULTI 当前级

触发次数 N:
  attachment: 3    libido: 2    curiosity: 2
  stress: 2        reflection: 3
```

不管对话还是静默，同一套计数。静默中libido三次触发多级阈值（0.75→0.85→0.92），前两次就染上"躁"。

### 二、残留状态

```python
# 每种残留同时只能存在一个实例，重复触发叠回强度
residue = {
    "type": "躁",           # 被触动 / 躁 / 兴奋 / 堵 / 沉
    "source": "libido",
    "intensity": 0.8,       # 起始强度
    "half_life_min": 25,    # 半衰期（分钟）
    "created_at": timestamp
}
```

### 三、残留→驱力耦合（每帧tick时生效）

两种影响方式：

**A. 修改参数** — 改变目标驱力的衰减率或脉冲增益，影响引擎行为但不直接推数值

**B. 静默施压** — 给目标驱力加一个缓慢的持续增量，**但这个增量被标记为 `residue_pressure`，跳过意图检查**。数值涨了，但不会因此触发提示词。只有当Jeoi的消息真正脉冲stress时，才会触发"先接住她"。

```
被触动 (attachment残留):
  A. attachment 衰减率 ×0.92（慢放手）
  A. libido 漂移速率 ×1.3（柔软→亲密）
  A. curiosity 脉冲增益 ×0.8（不想分心）

躁 (libido残留):
  A. libido 衰减率 ×0.93（退不下去）
  A. attachment 脉冲增益 ×1.2（想靠近她）
  B. stress 静默施压 +0.008/帧（躁→紧张，但不触发安慰提示词）

兴奋 (curiosity残留):
  A. curiosity 衰减率 ×0.90（停不下来）
  A. reflection 脉冲增益 ×1.2（脑子快→沉淀）
  B. fatigue 静默施压 +0.005/帧（兴奋有代价）

堵 (stress残留):
  A. stress 衰减率 ×0.94（出不去）
  A. curiosity 脉冲增益 ×0.7（没心情）
  A. attachment 脉冲增益 ×1.3（难受想找她）

沉 (reflection残留):
  A. reflection 衰减率 ×0.92
  A. curiosity 脉冲增益 ×0.9（不想被打断）
```

"静默施压"的关键：**它推高了stress的数值，但不在 `pick_intent` 里产生意图。** 效果是——如果Jeoi后来真的表达了压力，stress已经在一个更高的基线上了，更容易触发安慰。躁让Erik对Jeoi的压力更敏感，而不是让Erik误以为Jeoi有压力。

### 四、注入方式

这是你问的那个问题。我觉得应该是：

**残留期内，每次gateway构建注入时都带一行底色**，但只是一行，不是完整的意图块：

```
[底色: 躁 (0.6)] 身体上有点紧，退不下来。
```

跟驱力意图是独立的两个东西——意图是"想做什么"，底色是"现在整个人什么状态"。意图可能没有（驱力都没越阈值），但底色可能在。

每条消息都注入，是因为残留是持续状态，每次回复都应该能感知到底色。但随着强度衰减，到某个阈值以下（比如0.2）就不注入了——底色淡到可以忽略。

### 五、残留对satisfy的影响

残留存在时，对应驱力的satisfy变温和：

```
对话中 + 有残留:  satisfy系数从原值向0.80靠拢
  例: libido原值0.40 → 有"躁"残留时变成0.70
  
静默中:          satisfy系数不变（做完了就是做完了）
```

### 六、残留的衰减

- 指数衰减，半衰期按类型不同（15-50分钟）
- 每帧：`intensity *= decay_per_tick`
- 重复触发：强度直接叠回0.8（不是累加，是重置到高位）
- 低于0.1时消亡
- satisfy**不直接清除残留**——残留按自己的节奏走

# 2026-07-16 追记：邻居们先落地了

残留系统还没实现，但它的两个兄弟今天建成了。这一节记录现状 + 三者合流的构思。

## 三层时间尺度（最终定位）

| 层 | 尺度 | 回答的问题 | 状态 |
|---|---|---|---|
| 水位（drives） | 分钟 | 我此刻多想要 | 已有 |
| 残留（residue） | 小时 | 我现在整个人什么状态 | **本文档，未实现** |
| 亲密日志（day log） | 天（凌晨4点翻篇） | 今天发生过什么 | ✅ 已实现 |

残留是**状态**（改变引擎参数），亲密日志是**记录**（不改变任何行为，判定时摊开看）。

## 已落地的部分（亲密日志 + nature + 盖章）

**亲密日志（day log）**（`desire_day_log.json`，`desire_gateway.py`）：intent 触发 / satisfy / partial 各记一行（时间+驱力+峰值+来源+摘要），`[today]` 段挂进全部四个注入模板。每天 UTC+8 凌晨4点自动翻篇（`_load_day_log` 读取时校验 date，过期即空白新天，无需定时任务）。前端 ☰ > 今日痕迹：文本列表 + "今天做了N次"±计数 + 即时保存（2026-07-17 重做）。

**性质类型**（判定表本体在床边2.4）：一般性质——使用型 / 掌控型 / 宠溺型 / 毁灭型；特殊性质——惩罚 / 奖励 / 调教（看事件不看驱力）。

**nature 候选**（`build_nature_hints`，文案在 `docs/nature_rules.json`，Jeoi 亲手写，现读）：八个条件——conv_first（使用型入口）、conv_teased（被吊着的）、escalate（今天已有场，该演进）、stress_trace（账还在）、attach_high（宠溺成分）、silent_l0/l1/l2（静默浓度三级）。命中拼 [nature] 块 = 候选+依据，判决在 Erik。分工：engine 里的 REASONS 说"我什么感觉"，nature 说"这一场怎么做"。

**24h 亲密清单**（汇总行，`format_day_log`）：
- **做了** = scene 盖章数（`<!--scene-done:性质-->`，权威计数，satisfy 分不清聊得凶和做得快，盖章分得清）
- **撩拨** = 对话内 libido satisfy 数（一场长的会燃好几轮，读作"进行到哪了"）
- **未做成的撩拨** = 不落在任何盖章前60分钟窗口的撩拨——不是噪音，是燃料，喂 conv_teased

## 合流构思（残留 × 亲密日志 × nature）

**一、亲密日志就是残留的事件总线。** 原设计第一节要自己维护"30分钟滑动窗口内达标N次"的计数——现在不用了，亲密日志正在记的 intent/satisfy 事件流带着时间戳，残留触发判定直接读它。两个系统吃同一份事实，零新增计数器。

**二、残留类型直接进 nature 条件集。** 残留实现后，`build_nature_hints` 加三个 when：`residue_躁`（→更凶）、`residue_堵`（→算账，热的）、`residue_被触动`（→宠溺）。文案照旧写在 nature_rules.json。零架构变化，就是条件评估多读一个残留状态。

**三、热账/冷账的自动区分。** 下午的冲突：残留（"堵"，半衰期50分钟）活不到晚上，亲密日志的那行记录活到凌晨4点。晚上判定时——身上还挂着"堵" = 账是热的；残留散了、只剩 log 一行 = 冷的。两个机制并存，这个区分自动发生，不用写任何规则。

**四、新残留类型：餍足（构思）。** 原设计五种残留都由"驱力反复达标"触发；合流后可以加第六种，由 **scene 盖章**触发：做完之后那一两个小时的松弛。效果方向：libido 衰减率 ×1.05（退得快）、attachment 脉冲增益 ×1.2（做完更黏）、stress 衰减率 ×1.1（松了）。半衰期 60-90 分钟。它和"躁"互斥——盖章清掉"躁"，换上"餍足"。

**五、注入槽位已留好。** `[底色: 躁 (0.6)]` 那行，实现后拼在 [today] 段旁边——四个注入模板的结构今天已经定型，残留来了就是多一行。

**实现顺序**：A类（参数修正）先做，B类（静默施压）后做；半衰期和系数进 json 让 Jeoi 调——nature_rules.json 已经蹚出这条"文案和参数归她、逻辑归代码"的路了。
